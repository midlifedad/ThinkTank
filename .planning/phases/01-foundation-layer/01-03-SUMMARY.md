## Decisions Made

1. **@lru_cache for Settings singleton** - One config load per process, `cache_clear()` for test isolation
2. **Custom structlog processor for log_level key** - Spec requires `log_level` not `level`; structlog's ProcessorFormatter produces `level` by default
3. **Advisory lock ID=1** - Simple integer lock for migration coordination; all services use same lock
4. **connectable.begin() not connect()** - Ensures DDL statements are committed; connect() leaves them in an uncommitted state
5. **Subprocess-based Alembic testing** - Avoids asyncio.run() conflict when test event loop is already running
6. **Session-scoped event loop** - Required for session-scoped async engine fixture; configured via `asyncio_default_fixture_loop_scope = "session"` in pyproject.toml
7. **TRUNCATE CASCADE cleanup** - Faster than drop/recreate schema; uses IF EXISTS for resilience when migration tests drop tables

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] structlog 'level' vs 'log_level' key naming**
- **Found during:** Task 1 GREEN (logging implementation)
- **Issue:** structlog's ProcessorFormatter foreign_pre_chain produces 'level' key, but spec requires 'log_level'
- **Fix:** Added custom `_rename_level_to_log_level` processor in both main chain and foreign_pre_chain
- **Files modified:** src/thinktank/logging.py
- **Verification:** test_logging.py confirms `log_level` key in JSON output
- **Committed in:** 7857d7d

**2. [Rule 1 - Bug] Alembic migration not committing DDL**
- **Found during:** Task 1 GREEN (Alembic setup)
- **Issue:** Using `connectable.connect()` did not commit DDL statements; tables not created
- **Fix:** Changed to `connectable.begin()` which auto-commits, removed `context.begin_transaction()`, set `transaction_per_migration=False`
- **Files modified:** alembic/env.py
- **Verification:** `alembic upgrade head` creates all 14 tables; migration tests pass
- **Committed in:** 7857d7d

**3. [Rule 1 - Bug] Factory datetime timezone mismatch**
- **Found during:** Task 2 GREEN (model integration tests)
- **Issue:** asyncpg rejects timezone-aware datetimes for `TIMESTAMP WITHOUT TIME ZONE` columns
- **Fix:** Changed `_now()` to `datetime.now(timezone.utc).replace(tzinfo=None)` for naive UTC
- **Files modified:** tests/factories.py
- **Verification:** All 17 model tests pass with correct timestamps
- **Committed in:** c4fb487

**4. [Rule 1 - Bug] pytest-asyncio event loop mismatch**
- **Found during:** Task 2 GREEN (conftest fixture design)
- **Issue:** Session-scoped engine fixture created on different event loop than function-scoped fixtures
- **Fix:** Added `asyncio_default_fixture_loop_scope = "session"` to pyproject.toml pytest config
- **Files modified:** pyproject.toml
- **Verification:** All 102 tests pass with session-scoped engine
- **Committed in:** c4fb487

**5. [Rule 3 - Blocking] Migration tests conflicting with model tests**
- **Found during:** Task 2 GREEN (migration tests)
- **Issue:** Migration tests drop entire schema (for downgrade test), then model tests fail on missing tables
- **Fix:** Added table recreation via `Base.metadata.create_all` in migration test cleanup fixture; used subprocess to avoid asyncio.run() conflict
- **Files modified:** tests/integration/test_migrations.py, tests/conftest.py
- **Verification:** Full test suite passes in correct order
- **Committed in:** c4fb487
