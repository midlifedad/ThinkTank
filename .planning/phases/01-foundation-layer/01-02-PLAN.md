    Create all models following ThinkTank_Specification.md Section 3 exactly. Use SQLAlchemy 2.0 declarative style with `Mapped` and `mapped_column`.

    **Base and mixins** (`base.py`):
    - `Base(AsyncAttrs, DeclarativeBase)` -- AsyncAttrs enables async relationship access
    - `uuid_pk` annotated type: `Mapped[uuid.UUID]` with `mapped_column(primary_key=True, default=uuid.uuid4)`
    - `TimestampMixin` with `created_at: Mapped[datetime]` using `mapped_column(server_default=text("NOW()"))`
    - Use `from sqlalchemy.dialects.postgresql import JSONB, ARRAY` for PostgreSQL-specific types -- CRITICAL: do not use generic types for JSONB/ARRAY columns per pitfall #1 from research

    **14 models** (one module per logical group):

    1. `category.py` -- **Category** (spec 3.1) and **ThinkerCategory** (spec 3.3):
       - Category: id (UUID PK), slug (TEXT UNIQUE), name (TEXT), parent_id (UUID FK->categories, nullable), description (TEXT), created_at
       - Category.parent relationship (self-referential), Category.children relationship
       - ThinkerCategory: thinker_id (UUID FK->thinkers, PK), category_id (UUID FK->categories, PK), relevance (SMALLINT), added_at
       - Composite primary key on (thinker_id, category_id)

    2. `thinker.py` -- **Thinker** (spec 3.2), **ThinkerProfile** (spec 3.4), **ThinkerMetrics** (spec 3.5):
       - Thinker: id, name, slug (UNIQUE), tier (SMALLINT), bio, primary_affiliation, twitter_handle, wikipedia_url, personal_site, approval_status (TEXT, default="pending_llm"), approved_backfill_days (INT, nullable), approved_source_types (ARRAY(TEXT), nullable), active (BOOLEAN, default=True), added_at, last_refreshed (nullable)
       - Relationships: sources, profiles, metrics, categories (via ThinkerCategory)
       - ThinkerProfile: id, thinker_id (FK), education (JSONB), positions_held (JSONB), notable_works (JSONB), awards (JSONB), updated_at
       - ThinkerMetrics: id, thinker_id (FK), platform, handle, followers (BIGINT), avg_views (BIGINT), post_count, verified (BOOLEAN), snapshotted_at

    3. `source.py` -- **Source** (spec 3.6):
       - id, thinker_id (FK->thinkers), source_type, name, url (UNIQUE), external_id, config (JSONB, server_default="{}"), approval_status (default="pending_llm"), approved_backfill_days, backfill_complete (BOOLEAN, default=False), refresh_interval_hours, last_fetched (nullable), item_count (INT, default=0), active (BOOLEAN, default=True), error_count (INT, default=0), created_at
       - Relationship: thinker, content

    4. `content.py` -- **Content** (spec 3.7) and **ContentThinker** (spec 3.8):
       - Content: id, source_id (FK->sources), source_owner_id (FK->thinkers), content_type, url, canonical_url (UNIQUE), content_fingerprint (UNIQUE, nullable), title, body_text (nullable), word_count (nullable), published_at (nullable), duration_seconds (nullable), show_name (nullable), host_name (nullable), thumbnail_url (nullable), transcription_method (nullable), status (default="pending"), error_message (nullable), discovered_at, processed_at (nullable)
       - ContentThinker: content_id (FK, PK), thinker_id (FK, PK), role, confidence (SMALLINT), added_at
       - Composite PK on ContentThinker

    5. `candidate.py` -- **CandidateThinker** (spec 3.9):
       - id, name, normalized_name, appearance_count (INT, default=1), first_seen_at, last_seen_at, sample_urls (ARRAY(TEXT)), inferred_categories (ARRAY(TEXT)), suggested_twitter (nullable), suggested_youtube (nullable), status (default="pending_llm"), llm_review_id (FK->llm_reviews, nullable), reviewed_by (nullable), reviewed_at (nullable), thinker_id (FK->thinkers, nullable)

    6. `job.py` -- **Job** (spec 3.10):
       - id, job_type, payload (JSONB), status (default="pending"), priority (SMALLINT, default=5), attempts (SMALLINT, default=0), max_attempts (SMALLINT, default=3), error (nullable), error_category (nullable), last_error_at (nullable), worker_id (nullable), llm_review_id (FK->llm_reviews, nullable), scheduled_at (nullable), started_at (nullable), completed_at (nullable), created_at
       - Index on (status, priority, scheduled_at) for job claiming query

    7. `review.py` -- **LLMReview** (spec 3.11):
       - id, review_type, trigger, context_snapshot (JSONB), prompt_used, llm_response (nullable), decision (nullable), decision_reasoning (nullable), modifications (JSONB, nullable), flagged_items (JSONB, nullable), overridden_by (nullable), overridden_at (nullable), override_reasoning (nullable), model (nullable), tokens_used (INT, nullable), duration_ms (INT, nullable), created_at

    8. `config_table.py` -- **SystemConfig** (spec 3.12):
       - key (TEXT, primary_key=True -- NOT UUID), value (JSONB), set_by, updated_at
       - Note: SystemConfig uses TEXT PK (the config key), not UUID

    9. `rate_limit.py` -- **RateLimitUsage** (spec 3.13):
       - id, api_name, worker_id, called_at
       - Index on (api_name, called_at) for sliding-window queries

    10. `api_usage.py` -- **ApiUsage** (spec 3.14):
        - id, api_name, endpoint, period_start, call_count (INT), units_consumed (INT, nullable), estimated_cost_usd (Numeric(10,4), nullable)
        - Index on (api_name, period_start)

    **models/__init__.py**: Import and re-export ALL model classes so Alembic autogenerate can discover them. Also export Base.

    **Key constraints to get right:**
    - Use `Mapped[uuid.UUID]` for UUID columns -- SQLAlchemy 2.0 maps this to PostgreSQL UUID natively
    - Use `ARRAY(sa.Text)` from postgresql dialect for TEXT[] columns
    - Use `JSONB` from postgresql dialect for JSONB columns
    - All timestamps are `datetime` with `server_default=text("NOW()")` where the spec says "TIMESTAMPTZ"
    - Nullable columns use `Mapped[Optional[...]]` syntax
    - Use `relationship()` with `lazy="selectin"` for async-safe eager loading on key relationships (thinker.sources, source.content)
    - Indexes defined via `__table_args__` tuple or `index=True` on mapped_column
