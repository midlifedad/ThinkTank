<objective>
Build the llm_approval_check job handler that orchestrates LLM-gated approval for thinkers, sources, and candidates. Extend error categorization for Anthropic SDK exceptions. Register the handler and create integration + contract tests.

Purpose: This is the event-driven approval track of the LLM Supervisor (spec Section 8.1). When any entity enters `awaiting_llm` status, an `llm_approval_check` job is created. This handler builds context, calls the LLM, logs the audit trail, and applies the decision. It connects all Plan 01 building blocks into a working approval pipeline.

Output: Working `llm_approval_check` handler registered in the handler registry, with extended error categorization and full integration/contract test coverage.
</objective>

    1. Create `src/thinktank/handlers/llm_approval_check.py`:
       - Module-level `LLMClient` instance: `_llm_client = LLMClient()` (singleton, safe for concurrent async use per research).
       - Map review_type to (snapshot_builder, prompt_builder, response_schema):
         ```python
         REVIEW_TYPE_MAP = {
             "thinker_approval": (build_thinker_approval_context, build_thinker_approval_prompt, ThinkerApprovalResponse),
             "source_approval": (build_source_approval_context, build_source_approval_prompt, SourceApprovalResponse),
             "candidate_review": (build_candidate_review_context, build_candidate_review_prompt, CandidateReviewResponse),
         }
         ```
       - `async def handle_llm_approval_check(session: AsyncSession, job: Job) -> None`:
         1. Extract `review_type = job.payload.get("review_type")` and `target_id = uuid.UUID(job.payload["target_id"])`. Raise ValueError if missing.
         2. Look up (snapshot_builder, prompt_builder, response_schema) from REVIEW_TYPE_MAP. Raise ValueError if unknown review_type.
         3. For candidate_review: `target_id` may be None (batch mode), use `candidate_ids` from payload instead.
         4. Build context: `context = await snapshot_builder(session, target_id)` (or candidate_ids for batch).
         5. Build prompts: `system_prompt, user_prompt = prompt_builder(context)`.
         6. Call LLM: `result, tokens_used, duration_ms = await _llm_client.review(system_prompt, user_prompt, response_schema)`.
         7. Create LLMReview row with all fields: review_type, trigger="job_gate", context_snapshot=context, prompt_used=f"{system_prompt}\n\n{user_prompt}", llm_response=result.model_dump_json(), decision=result.decision, decision_reasoning=result.reasoning, modifications=result.modifications (if exists), flagged_items=result.flagged_items (if exists), model=_llm_client.model, tokens_used=tokens_used, duration_ms=duration_ms. Add to session, flush to get review.id.
         8. Extract `pending_job_id` from payload (optional).
         9. Call `await apply_decision(session, review_type, target_id, pending_job_id, result, review.id)`.
         10. Commit the session.
         11. Log with structlog: "llm_approval_completed", review_type, target_id, decision, tokens_used.
       - Let anthropic exceptions propagate to the worker loop for categorization and retry.

         ```python
         import anthropic

         if isinstance(exc, pydantic.ValidationError):

         ```
