    1. Create `src/thinktank/llm/time_utils.py`:
       - `def seconds_until_next_utc_hour(target_hour: int) -> float`: Compute seconds until the next occurrence of target_hour UTC. Uses `datetime.now(UTC)`, replaces hour/minute/second/microsecond. If target <= now, add timedelta(days=1). Returns `(target - now).total_seconds()`.
       - `def seconds_until_next_monday_utc(target_hour: int) -> float`: Compute seconds until next Monday at target_hour UTC. Handles "already past Monday target_hour" case by going to next Monday. Uses `now.weekday()` (0=Monday).

    2. Create `src/thinktank/llm/escalation.py`:
       - `async def escalate_timed_out_reviews(session: AsyncSession) -> int`: Per spec Section 8.6.
         - Read `llm_timeout_hours` from system_config via `get_config_value(session, "llm_timeout_hours", 2)`.
         - Execute raw SQL UPDATE (matching the project pattern from reclaim.py):
           ```sql
           UPDATE jobs
           SET payload = jsonb_set(
               COALESCE(payload, '{}'::jsonb),
               '{needs_human_review}',
               'true'::jsonb
           )
           WHERE status = 'awaiting_llm'
             AND created_at < LOCALTIMESTAMP - MAKE_INTERVAL(hours => :timeout_hours)
             AND NOT COALESCE((payload->>'needs_human_review')::boolean, false)
           RETURNING id, job_type
           ```
         - For each escalated job, create an `LLMReview` row with: review_type="timeout_escalation", trigger="scheduled", context_snapshot={"job_id": str(id), "job_type": job_type}, prompt_used="N/A - timeout escalation", decision="escalate_to_human", decision_reasoning=f"LLM API unavailable for >{timeout_hours}h. Escalated to human review.".
         - Flush session. Return count of escalated jobs.
       - Use `from sqlalchemy import text` for the raw SQL (matching existing reclaim.py pattern).

    3. Create `src/thinktank/llm/scheduled.py`:
       - Module-level `_llm_client = LLMClient()` singleton.
       - `async def run_health_check(session: AsyncSession) -> LLMReview | None`:
         1. Build context: `context = await build_health_check_context(session)`.
         2. Build prompts: `system_prompt, user_prompt = build_health_check_prompt(context)`.
         3. Call LLM: `result, tokens, duration = await _llm_client.review(system_prompt, user_prompt, HealthCheckResponse)`.
         4. Create LLMReview: review_type="health_check", trigger="scheduled", context_snapshot=context, prompt_used, llm_response=result.model_dump_json(), decision=result.status, decision_reasoning="; ".join(result.findings), model, tokens_used, duration_ms.
         5. If result.config_adjustments: apply safe config updates (update system_config rows within bounds). Log adjustments.
         6. Return the review.
       - `async def run_daily_digest(session: AsyncSession) -> LLMReview | None`: Same pattern with DailyDigestResponse. review_type="daily_digest". Log as summary.
       - `async def run_weekly_audit(session: AsyncSession) -> LLMReview | None`: Same pattern with WeeklyAuditResponse. review_type="weekly_audit". Log recommendations.
       - All three catch `Exception` (not just anthropic errors) and log via structlog. Return None on failure. Do NOT re-raise -- scheduled tasks must not crash the scheduler.
