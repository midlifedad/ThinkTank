<objective>
Wire the discovery modules from Plan 01 into three job handlers that the worker loop dispatches, register them in the handler registry, extend error categories, and validate with integration and contract tests.

Purpose: These handlers are the runtime entry points for autonomous growth. scan_for_candidates feeds the LLM approval pipeline with new candidates. discover_guests_listennotes and discover_guests_podcastindex register new podcast sources for approved thinkers. Together they close the discovery loop: content ingestion -> name extraction -> candidate creation -> LLM approval -> thinker activation -> guest discovery -> source registration -> more content ingestion.

Output: 3 new handlers registered in the worker loop, extended error categories, integration tests proving the full discovery flow, and contract tests verifying handler side effects.

  <behavior>
    scan_for_candidates handler:
    - Reads content_ids from job.payload, extracts names from each content item's title + description
    - Skips names that match existing thinkers (via find_similar_thinkers trigram)
    - For names matching existing candidates (via find_similar_candidates trigram): increments appearance_count and updates last_seen_at
    - For new names: creates CandidateThinker with normalized_name, status=pending_llm, appearance_count=1
    - Names with appearance_count reaching 3 get status=pending_llm (already default)
    - Checks daily quota before creating candidates; stops if quota exhausted
    - If should_trigger_llm_review returns True: enqueues llm_approval_check job with review_type=candidate_review
    - If pending_llm queue > 40: logs warning and returns early (cascade pause)

    discover_guests_listennotes handler:
    - Reads thinker_id from job.payload, loads thinker name
    - Searches Listen Notes for episodes featuring the thinker
    - For each result with a podcast.rss URL: checks if source already exists (url_normalizer), if not creates Source with approval_status=pending_llm linked to thinker
    - If rate-limited (client returns None): raises ValueError to trigger job retry

    discover_guests_podcastindex handler:
    - Reads thinker_id from job.payload, loads thinker name
    - Searches Podcast Index for episodes by person
    - For each result with feedUrl: checks if source already exists, if not creates Source pending LLM approval
    - If rate-limited: raises ValueError to trigger retry

    Error categories:
    - LISTENNOTES_RATE_LIMIT added to ErrorCategory enum
    - PODCASTINDEX_ERROR added to ErrorCategory enum
    - categorize_error extended to catch httpx.HTTPStatusError with 429 status as LISTENNOTES_RATE_LIMIT
