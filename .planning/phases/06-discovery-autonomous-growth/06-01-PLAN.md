       - Import `normalize_name` from `src.thinktank.ingestion.name_normalizer`.
       - Define `_GUEST_PATTERNS` list of compiled regexes for common podcast guest name formats:
         - `(?:with|w/)\s+([A-Z][a-z]+(?:\s+[A-Z][a-z]+)+)` -- "with John Smith" / "w/ John Smith"
         - `(?:feat\.?|featuring)\s+([A-Z][a-z]+(?:\s+[A-Z][a-z]+)+)` -- "feat. Jane Doe"

         - `#?\d+\s*[-\u2013\u2014:]\s*([A-Z][a-z]+(?:\s+[A-Z][a-z]+)+)` -- "#123 - John Smith"
         - `\|\s*([A-Z][a-z]+(?:\s+[A-Z][a-z]+)+)` -- "| Sam Harris"
       - Define `_BLOCKLIST` set: {"the", "inc", "llc", "university", "foundation", "institute", "network", "podcast", "show", "episode", "series", "season", "part", "chapter", "volume"}
       - Define `_looks_like_person_name(name: str) -> bool`: 2-4 words, each word >= 2 chars, no all-caps words (len > 1), no blocklist words (lowercased).
       - Define `extract_names(title: str, description: str) -> list[str]`:
         - Iterate both title and description through all patterns.
         - For each regex match group(1), strip, validate with `_looks_like_person_name`.
         - If valid, normalize with `normalize_name()` and add to a set.
         - Return sorted list from the set (sorted for deterministic output).
       - This is a **pure function** -- no I/O, no async, no database.
