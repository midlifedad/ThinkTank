Add 4 new partial endpoints to the existing dashboard.py router (keep ALL existing endpoints unchanged):

1. **GET /admin/partials/health-summary** -- Returns health indicators:
   - Worker status: read `workers_active` from system_config (true/false). Display as green "ACTIVE" or red "STOPPED".
   - DB connection: execute `SELECT 1` in a try/except, display green "OK" or red "ERROR".
   - Error rate: count failed jobs in the last hour (`SELECT COUNT(*) FROM jobs WHERE status='failed' AND created_at > NOW() - INTERVAL '1 hour'`). Display count with red color if > 0.
   - Template: `partials/health_summary.html` -- use stat-row divs with color classes (.healthy/.error) matching base.html CSS.

2. **POST /admin/kill-switch/toggle** -- Toggle the kill switch:
   - Read current `workers_active` from system_config.
   - If exists, flip the boolean value. If not exists, create with value `false` (turning off).
   - Write back with `set_by = "admin"`.
   - Commit, then re-render the kill_switch partial with updated state.
   - Use `from sqlalchemy import select` with SystemConfig model for the read/write (not raw SQL) to match the api_keys.py pattern.

3. **GET /admin/partials/kill-switch** -- Render kill switch state:
   - Read `workers_active` from system_config.
   - Template: `partials/kill_switch.html` -- show a prominent toggle button. If active: green background, text "Workers ACTIVE", button says "Stop Workers". If inactive: red background, text "Workers STOPPED", button says "Start Workers". The button uses `hx-post="/admin/kill-switch/toggle" hx-target="#kill-switch-container" hx-swap="innerHTML"`.

4. **GET /admin/partials/activity-feed** -- Returns last 50 system actions:
   - Use a UNION ALL query across multiple tables to build a unified activity feed:
     ```sql
     (SELECT 'job_completed' as action_type, job_type as detail, completed_at as action_time
      FROM jobs WHERE status = 'complete' AND completed_at IS NOT NULL ORDER BY completed_at DESC LIMIT 20)
     UNION ALL
     (SELECT 'job_failed' as action_type, job_type as detail, last_error_at as action_time
      FROM jobs WHERE status = 'failed' AND last_error_at IS NOT NULL ORDER BY last_error_at DESC LIMIT 15)
     UNION ALL
     (SELECT 'llm_decision' as action_type, review_type || ': ' || COALESCE(decision, 'pending') as detail, created_at as action_time
      FROM llm_reviews WHERE decision IS NOT NULL ORDER BY created_at DESC LIMIT 15)
     ORDER BY action_time DESC LIMIT 50
     ```
   - Template: `partials/activity_feed.html` -- compact list with icon/emoji per action type, detail text, and relative time. Use a scrollable div with max-height: 400px.

5. **GET /admin/partials/pending-approvals** -- Returns pending count:
   - `SELECT COUNT(*) FROM llm_reviews WHERE decision IS NULL`
   - Template: `partials/pending_approvals.html` -- display count as a badge number. If > 0, show in orange/warning. Link to /admin/llm for details.

Create all 4 partial templates following the existing pattern: minimal HTML fragments, no extending base.html, use existing CSS classes from base.html (stat-row, stat-label, stat-value, healthy, error, warning, btn, etc.).
