**Create `src/thinktank/admin/routers/config.py`** with the following endpoints:

1. **GET /admin/config** -- Config landing page:
   - Render `config.html` with links/sections for: Rate Limits, System Settings, API Keys (/admin/api-keys), Categories (/admin/categories).
   - The rate limits and system settings sections use HTMX to load their partials inline.

2. **GET /admin/config/partials/rate-limits** -- Rate limits editor partial:
   - Load current limits from system_config key `rate_limits` (JSONB dict). Fall back to defaults: `{"youtube": 200, "podcastindex": 500, "anthropic": 50}`.
   - Template: `partials/rate_limits_editor.html` -- render an editable form with one number input per API. Each input has `name="limit_{api_name}"` and `type="number" min="1"`.
   - Form submits via `hx-post="/admin/config/rate-limits/save" hx-target="#rate-limits-editor" hx-swap="innerHTML"`.

3. **POST /admin/config/rate-limits/save** -- Save rate limits:
   - Read form values `limit_youtube`, `limit_podcastindex`, `limit_anthropic` via `Form(...)`.
   - Validate all are positive integers.
   - Upsert system_config row with key `rate_limits`, value as dict `{"youtube": int, "podcastindex": int, "anthropic": int}`, set_by "admin".
   - Use `select(SystemConfig).where(SystemConfig.key == 'rate_limits')` pattern from api_keys.py.
   - On success, re-render the rate limits editor partial with updated values and a success flash message.

4. **GET /admin/config/partials/system-settings** -- System config editor partial:
   - Load the following config keys from system_config (with sensible defaults if not set):
     - `llm_timeout_hours` (default: 2) -- "LLM Timeout (hours)"
     - `backpressure_threshold` (default: 100) -- "Backpressure Threshold (queue depth)"
     - `stale_job_minutes` (default: 30) -- "Stale Job Timeout (minutes)"
     - `max_candidates_per_day` (default: 50) -- "Max Candidates Per Day"
   - For each, the value in system_config may be stored as a raw int, a JSONB number, or a JSONB dict with a "value" key -- handle all cases by coercing to int.
   - Template: `partials/system_config_editor.html` -- render a form with labeled number inputs for each setting. Form submits via `hx-post="/admin/config/system/save" hx-target="#system-config-editor" hx-swap="innerHTML"`.

5. **POST /admin/config/system/save** -- Save system config:
   - Read form values for each setting via Form(...).
   - For each key, upsert the system_config row. Store value as the raw integer (JSONB), set_by "admin", updated_at now.
   - Commit all in one transaction.
   - Re-render the system config editor partial with updated values and a success flash.
