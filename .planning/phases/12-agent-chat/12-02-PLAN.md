**C. JavaScript** (add a script block before closing body tag):

Implement these functions:

- `getChatState()` / `saveChatState(state)`: Read/write `{sessionId, messages, isOpen}` from/to localStorage key `thinktank_chat_state`. Cap messages at 100 (trim oldest when exceeded).
- `toggleChat()`: Toggle `.chat-collapsed` class on `#chat-drawer`, save isOpen state, focus input if opening.
- `renderMessages()`: Clear `#chat-messages`, iterate stored messages, call appropriate append function for each (user, assistant, proposal, error). Scroll to bottom.
- `appendUserMessage(text, save)`: Create div with `.chat-msg .chat-msg-user`, set textContent, append to `#chat-messages`, optionally save to localStorage.
- `appendAssistantMessage(text, save)`: Same pattern with `.chat-msg-assistant`.
- `appendErrorMessage(text, save)`: Same with `.chat-msg-error`, prefix "Error: ".
- `appendProposal(proposal, save)`: Create div with `.chat-proposal`, id `proposal-{proposal_id}`. Render: bold action_type, target text, explanation in small, and two buttons (Confirm calling `confirmProposal(id)`, Dismiss calling `dismissProposal(id)`). Use innerHTML with escapeHtml for text content.
- `escapeHtml(text)`: Create a temp div, set textContent, return innerHTML.
- `sendMessage(event)`: preventDefault, get input value, clear input, disable send button, call appendUserMessage. Create a streaming placeholder div (`#streaming-msg`, class includes `.chat-msg-streaming`). Build FormData with `message` and optional `session_id`. Use `fetch('/admin/chat/send', {method: 'POST', body: formData})` then read response via `response.body.getReader()`. Parse SSE events manually from chunks (split on newlines, look for `data: ` prefix, JSON.parse). Handle event types: `session_init` (save sessionId), `text_delta` (append to placeholder textContent), `proposal` (call appendProposal), `error` (call appendErrorMessage), `done` (no-op). On stream complete: remove streaming class from placeholder, save final assistant message to localStorage. Re-enable send button.
  IMPORTANT: Use fetch() with ReadableStream, NOT EventSource. EventSource only supports GET but our endpoint is POST.
- `confirmProposal(proposalId)`: Build FormData with session_id, POST to `/admin/chat/confirm/{proposalId}`, parse JSON response. Replace proposal div content with success (green) or error (red) message.
- `dismissProposal(proposalId)`: Replace proposal div content with "Action dismissed" (gray).
- DOMContentLoaded listener: Restore chat state (open/closed), call renderMessages().
