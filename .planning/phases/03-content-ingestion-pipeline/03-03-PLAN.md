    1. Create `src/thinktank/ingestion/config_reader.py` with two functions:

    `async def get_config_value(session: AsyncSession, key: str, default: Any) -> Any` -- Query system_config table for key, return value (JSONB) or default. Uses `select(SystemConfig.value).where(SystemConfig.key == key)`.

    `def get_source_filter_config(source_config: dict, global_min_duration: int, global_skip_patterns: list[str]) -> tuple[int, list[str]]` -- Pure function. Returns (effective_min_duration, effective_skip_patterns) by checking source.config for `min_duration_override`, `skip_title_patterns_override`, `additional_skip_patterns`. If `min_duration_override` present, use it; otherwise use global. If `skip_title_patterns_override` present, use it; otherwise use global + any `additional_skip_patterns`.

    2. Create `src/thinktank/handlers/fetch_podcast_feed.py` with `async def handle_fetch_podcast_feed(session: AsyncSession, job: Job) -> None`.

    Handler logic (thin orchestrator per RESEARCH.md Pattern 2):

    a. Extract `source_id` from `job.payload["source_id"]`. Raise ValueError if missing.
    b. Load source: `await session.get(Source, source_id)`. Raise ValueError if not found.
    c. Verify source is approved and active: `source.approval_status == "approved" and source.active`. If not, log warning and return (no error -- just skip).
    d. Load source owner thinker: `await session.get(Thinker, source.thinker_id)`.
    e. Read global config values: `min_duration_seconds` (default 600), `skip_title_patterns` (default list from spec). Use get_config_value().
    f. Compute effective filter config from source.config overrides via get_source_filter_config().
    g. Fetch the RSS feed: Use `httpx.AsyncClient` with 60s timeout. `response = await client.get(source.url, timeout=60.0)`. Raise on non-200 status.
    h. Parse feed: `entries = parse_feed(response.text)`.
    i. Determine if backfill or incremental: If `source.backfill_complete == False`, process all entries. If True, only process entries where `entry.published_at > source.last_fetched` (skip entries published before last_fetched).
    j. For each entry:
       - Normalize URL: `canonical = normalize_url(entry.url)`
       - Check URL dedup (Layer 1): `SELECT 1 FROM content WHERE canonical_url = :canonical`. If exists, skip.
       - Compute fingerprint: `fp = compute_fingerprint(entry.title, entry.published_at, entry.duration_seconds)`
       - Check fingerprint dedup (Layer 2): If fp is not None, `SELECT 1 FROM content WHERE content_fingerprint = :fp`. If exists, log alias URL and skip.
       - Determine status: If `should_skip_by_duration(entry.duration_seconds, effective_min_duration)` or `should_skip_by_title(entry.title, effective_skip_patterns)`, set status='skipped'. Otherwise status='pending'.
       - Insert Content row with all fields. Use `session.add(Content(...))`.
       - Track count of inserted items.
       - Collect entry descriptions for attribution: build a dict mapping `str(content.id)` to `entry.description or ""` for each inserted content row.
    k. After processing all entries:
       - Update `source.last_fetched = _now()` (use the timezone-naive _now() pattern).
       - Update `source.item_count += inserted_count`.
       - If this was backfill mode, set `source.backfill_complete = True`.
       - Enqueue a `tag_content_thinkers` job for the batch: create a Job with `job_type="tag_content_thinkers"`, `payload={"content_ids": [str(c.id) for c in inserted], "source_id": str(source_id), "descriptions": descriptions_dict}`, `priority=3`. **IMPORTANT:** The `descriptions` dict maps content_id (str) to description text (str). This is the canonical payload schema for tag_content_thinkers -- Plan 04's handler reads descriptions from this dict to match thinker names in episode descriptions (confidence=6).
    l. Commit the session (all content inserts + source updates in one transaction).
    m. Use structlog for logging throughout: log source name, entry count, inserted count, skipped count, dedup count.
    n. Wrap httpx calls in try/except: TimeoutError -> re-raise (worker loop categorizes as HTTP_TIMEOUT). httpx.HTTPStatusError -> re-raise as appropriate. feedparser errors -> re-raise ValueError (categorized as RSS_PARSE).

    3. Create `src/thinktank/handlers/refresh_due_sources.py` with `async def handle_refresh_due_sources(session: AsyncSession, job: Job) -> None`.

    Handler logic:
    a. Query for sources due for refresh using raw SQL via `text()` (per RESEARCH.md):
       ```sql
       SELECT id FROM sources
       WHERE active = true
         AND approval_status = 'approved'
         AND (
             last_fetched IS NULL
             OR last_fetched + MAKE_INTERVAL(hours => refresh_interval_hours) < LOCALTIMESTAMP
         )
       ```
    b. For each due source, create a `fetch_podcast_feed` job:
       ```python
       Job(job_type="fetch_podcast_feed", payload={"source_id": str(source_id)}, priority=2, status="pending")
       ```
    c. Log the count of jobs created.
    d. Commit.

    4. Update `src/thinktank/handlers/registry.py`:
    - Import the two handlers at module level (lazy import to avoid circular deps if needed).
    - Register both handlers after the JOB_HANDLERS dict definition:
      ```python
      from src.thinktank.handlers.fetch_podcast_feed import handle_fetch_podcast_feed
      from src.thinktank.handlers.refresh_due_sources import handle_refresh_due_sources

      register_handler("fetch_podcast_feed", handle_fetch_podcast_feed)
      register_handler("refresh_due_sources", handle_refresh_due_sources)
      ```
    - If circular import issues occur, use lazy registration in a `register_all()` function called from worker startup instead.
