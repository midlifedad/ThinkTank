  <action>
    Create `src/thinktank/queue/claim.py` with three async functions:

    `claim_job(session: AsyncSession, worker_id: str, job_types: list[str] | None = None) -> Job | None`:
    - SELECT from jobs WHERE status IN ('pending', 'retrying') AND (scheduled_at IS NULL OR scheduled_at <= NOW())
    - ORDER BY priority ASC, scheduled_at ASC (NULLS FIRST for scheduled_at)
    - with_for_update(skip_locked=True).limit(1)
    - If job_types provided, add WHERE job_type IN (job_types)
    - On claim: set status='running', worker_id=worker_id, started_at=_now(), attempts+=1
    - Commit immediately to release the FOR UPDATE lock
    - Return the claimed Job or None

    `complete_job(session: AsyncSession, job_id: uuid.UUID) -> None`:
    - UPDATE jobs SET status='done', completed_at=now, error=None, error_category=None WHERE id=job_id
    - Use ORM update statement, commit

    `fail_job(session: AsyncSession, job_id: uuid.UUID, error_msg: str, error_category: ErrorCategory, max_attempts: int | None = None) -> None`:
    - Fetch job by ID
    - If max_attempts not provided, use get_max_attempts(job.job_type)
    - If job.attempts < max_attempts: set status='retrying', scheduled_at=now + calculate_backoff(job.attempts), worker_id=None
    - Else: set status='failed', completed_at=now
    - Set error=error_msg, error_category=error_category.value, last_error_at=now
    - Commit
