    - get_effective_priority demotes by +3 (capped at 10) when process_content depth > threshold
    - get_effective_priority returns normal priority when depth < 80% of threshold
    - In the hysteresis band (80%-100%), returns original priority (no change)
    - BACKPRESSURE_JOB_TYPES includes all 10 discovery/fetch types from spec
    - Threshold read from system_config key "max_pending_transcriptions" (default 500)
  </behavior>

    - Return the integer value, or None if not found

    `check_and_acquire_rate_limit(session: AsyncSession, api_name: str, worker_id: str, window_minutes: int = 60) -> bool`:
    - COUNT(*) from rate_limit_usage WHERE api_name = api_name AND called_at > NOW() - interval

    `get_queue_depth(session: AsyncSession, job_type: str) -> int`:
    - COUNT(*) from jobs WHERE job_type = job_type AND status IN ('pending', 'retrying')

    `get_effective_priority(session: AsyncSession, job: Job) -> int`:
    - If job.job_type not in BACKPRESSURE_JOB_TYPES, return job.priority unchanged
    - Get depth via get_queue_depth(session, "process_content")
    - Get threshold from SystemConfig key "max_pending_transcriptions" (default 500 if missing)
    - If depth > threshold: return min(job.priority + 3, 10)
    - If depth < threshold * 0.8: return job.priority
    - Else (hysteresis band): return job.priority

    Create `src/thinktank/queue/reclaim.py` with:

    `reclaim_stale_jobs(session: AsyncSession) -> list[dict]`:
    - Use raw SQL via text() for this bulk operation (per research recommendation)
    - Get stale_job_timeout_minutes from system_config (default 30 if missing)
    - UPDATE jobs SET:
      - status = CASE WHEN attempts + 1 >= max_attempts THEN 'failed' ELSE 'retrying' END
      - worker_id = NULL
      - attempts = attempts + 1
      - error = 'Reclaimed: exceeded stale_job_timeout_minutes'
      - error_category = 'worker_timeout'
      - last_error_at = NOW()
      - scheduled_at = CASE WHEN attempts + 1 >= max_attempts THEN NULL ELSE NOW() + (POWER(2, attempts + 1) * INTERVAL '1 minute') END
      - completed_at = CASE WHEN attempts + 1 >= max_attempts THEN NOW() ELSE NULL END
    - WHERE status = 'running' AND started_at < NOW() - (timeout * INTERVAL '1 minute')
    - RETURNING id, job_type, worker_id, attempts, max_attempts
    - Return list of dicts from RETURNING rows
    - Commit after execution
