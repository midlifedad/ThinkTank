Create a new router at `src/thinktank/admin/routers/pipeline.py` with prefix `/admin/pipeline`.

**Endpoints to implement:**

1. `GET /admin/pipeline/` -- Pipeline page. Renders pipeline.html. No data loading -- the job list is loaded via HTMX partial on page load.

2. `GET /admin/pipeline/partials/jobs` -- HTMX partial: paginated, filtered job list. Accept query params:
   - `status` (optional): filter by job status. Values: "pending", "running", "failed", "done", "" (all). Note: the Job model uses "done" not "complete" for completed jobs (check claim.py complete_job which sets status="done").
   - `job_type` (optional): filter by job_type string. Values: any of the registered types or "" (all).
   - `date_from` (optional): ISO date string, filter jobs created_at >= this date.
   - `date_to` (optional): ISO date string, filter jobs created_at < this date + 1 day.
   - `page` (optional, default 1): page number for pagination.
   Build a SQLAlchemy query on Job, apply filters if provided (skip empty strings), order by created_at DESC, paginate with LIMIT 25 OFFSET (page-1)*25. Also run a COUNT query with same filters to compute total_pages. Return context: jobs list, page, total_pages, filters (to preserve in UI). Render partials/job_list.html.

3. `GET /admin/pipeline/jobs/{job_id}` -- Job detail. Load Job by ID (return 404 if not found). Render partials/job_detail.html with all job fields: id, job_type, status, priority, payload (as formatted JSON string), attempts, max_attempts, error, error_category, worker_id, scheduled_at, started_at, completed_at, created_at, last_error_at.

4. `POST /admin/pipeline/trigger/{job_type}` -- Manual job trigger. Accept `job_type` as path param. Validate job_type is one of: "refresh_due_sources", "scan_for_candidates", "discover_guests_podcastindex". For "discover_guests_podcastindex", accept optional Form field `thinker_id` (str UUID). Create a Job row: job_type=job_type, payload={"triggered_by": "admin"} (add thinker_id to payload if provided), status="pending", priority=5. Commit. Render partials/trigger_result.html with success message "Job {job_type} queued successfully" and the new job ID.

5. `POST /admin/pipeline/jobs/{job_id}/retry` -- Retry a failed job. Load the Job by ID (404 if not found). Verify status is "failed" (return error message if not). Create a NEW Job row copying job_type and payload from the original, status="pending", priority=5. Do NOT modify the original job. Commit. Return the job_list partial (redirect to list via hx-get or render inline success + re-trigger list load via HX-Trigger response header).

6. `POST /admin/pipeline/jobs/{job_id}/cancel` -- Cancel a pending job. Load the Job by ID (404 if not found). Verify status is "pending" (return error message if not). Update job.status = "cancelled". Commit. Return the job_list partial or trigger list reload.

**For retry and cancel responses:** Set the HX-Trigger response header to "refreshJobList" so the job list partial auto-refreshes after the action. The job_list container div should listen for this event: `hx-trigger="load, refreshJobList from:body"`.
