**Scheduler data model -- stored in system_config table:**

Each recurring task gets its own system_config row with key pattern `scheduler_{task_name}`. The JSONB value stores:
```json
{
  "frequency_hours": 6,
  "enabled": true,
  "last_run_at": "2026-03-10T12:00:00",
  "next_run_at": "2026-03-10T18:00:00"
}
```

**Define the list of configurable scheduled tasks as a constant in the router:**
```python
SCHEDULED_TASKS = [
    {"key": "refresh_due_sources", "label": "Refresh Due Sources", "default_hours": 1, "job_type": "refresh_due_sources"},
    {"key": "scan_for_candidates", "label": "Scan for Candidates", "default_hours": 24, "job_type": "scan_for_candidates"},
    {"key": "llm_health_check", "label": "LLM Health Check", "default_hours": 6, "job_type": None},
    {"key": "llm_daily_digest", "label": "LLM Daily Digest", "default_hours": 24, "job_type": None},
    {"key": "llm_weekly_audit", "label": "LLM Weekly Audit", "default_hours": 168, "job_type": None},
]
```

Note: Tasks with `job_type=None` (LLM tasks) are tracked for visibility but their Run Now creates no job -- the worker loop handles them via its own internal scheduler. For Run Now on LLM tasks, display a note "LLM tasks are managed by the worker loop" instead of creating a job.

**New endpoints to add:**

1. `GET /admin/pipeline/partials/scheduler` -- Scheduler editor partial. For each task in SCHEDULED_TASKS:
   - Read system_config row `scheduler_{task.key}` (SELECT WHERE key = ...)
   - If not found, use defaults: frequency_hours=task.default_hours, enabled=True, last_run_at=None, next_run_at=None
   - Build a list of task configs with: key, label, frequency_hours, enabled, last_run_at, next_run_at, has_job_type (bool)
   - Render partials/scheduler_editor.html with tasks list

2. `POST /admin/pipeline/scheduler/{task_key}/save` -- Save schedule config. Accept Form fields: `frequency_hours` (int). Validate task_key is in SCHEDULED_TASKS keys. Read existing system_config row for `scheduler_{task_key}`:
   - If exists: update value.frequency_hours, update value.next_run_at to last_run_at + new frequency (or now + new frequency if last_run_at is None), set set_by="admin", updated_at=utcnow()
   - If not exists: create with defaults + the specified frequency_hours
   - Commit. Re-render scheduler partial with success message.

3. `POST /admin/pipeline/scheduler/{task_key}/toggle` -- Toggle enabled/disabled. Read existing config, flip enabled boolean, recalculate next_run_at if being enabled (set to now + frequency_hours). Commit. Re-render scheduler partial.

4. `POST /admin/pipeline/scheduler/{task_key}/run-now` -- Run Now button. If the task has a job_type (refresh_due_sources, scan_for_candidates): create a Job row with that job_type, status="pending", payload={"triggered_by": "admin_scheduler_run_now"}, priority=5. Also update the system_config: set last_run_at=now, next_run_at=now+frequency. Commit. Re-render scheduler partial with success indicator.
   If the task has no job_type (LLM tasks): return the scheduler partial with an info message "LLM tasks run on their internal schedule in the worker loop".
