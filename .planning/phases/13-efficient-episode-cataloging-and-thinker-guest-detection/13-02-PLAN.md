<threat_model>
## Attack Surfaces

1. **YouTube API key exposure in logs**
   - Threat: API key leaked to structured logs
   - Severity: medium
   - Mitigation: YouTubeClient accepts api_key as constructor param; never log it. structlog bind does NOT include api_key.

2. **YouTube API quota exhaustion**
   - Threat: Runaway pagination exhausts daily 10,000 unit quota
   - Severity: medium
   - Mitigation: YouTubeClient has a `max_pages` parameter (default 100, = 5000 videos). Handler checks rate limit via `check_and_acquire_rate_limit(session, "youtube", ...)` before each API call. Handler tracks quota units consumed and stops if approaching limit.

3. **Malicious video metadata injection**
   - Threat: YouTube video titles/descriptions with XSS payloads stored in DB
   - Severity: low
   - Mitigation: Content stored as text in PostgreSQL; rendered via Jinja2 autoescaping in admin templates (already enabled project-wide). No raw HTML rendering.

**ASVS L1 applicable:** V2 (Authentication) -- API key management; V5 (Input Validation) -- URL validation for YouTube URLs.
</threat_model>

```python
class YouTubeClient:
    """YouTube Data API v3 client for channel video cataloging.

    Uses quota-efficient endpoints:
    - playlistItems.list: 1 quota unit per page (50 items)
    - videos.list: 1 quota unit per batch (50 video IDs)
    NOT search.list (100 units per call).
    """

    def __init__(self, api_key: str):
        from googleapiclient.discovery import build
        self._youtube = build("youtube", "v3", developerKey=api_key)
        self._quota_used = 0

    @property
    def quota_used(self) -> int:
        return self._quota_used

    def get_uploads_playlist_id(self, channel_id: str) -> str:
        """Convert channel ID to uploads playlist ID.
        Shortcut: replace 'UC' prefix with 'UU'. Costs 0 quota units.
        Fallback: channels.list API call (3 quota units) if prefix doesn't match.
        """

    def list_playlist_videos(self, playlist_id: str, page_token: str | None = None) -> dict:
        """Fetch a page of videos from a playlist.
        Uses playlistItems.list -- 1 quota unit per call.
        Returns raw API response dict with 'items' and optional 'nextPageToken'.
        """

    def get_video_details(self, video_ids: list[str]) -> list[dict]:
        """Fetch video details (duration, description, categoryId) in batches of 50.
        Uses videos.list -- 1 quota unit per call.
        Returns list of video detail dicts.
        """

    def fetch_all_channel_videos(self, channel_id: str, max_pages: int = 100) -> list[dict]:
        """Convenience: fetch ALL videos from a channel with details.
        Paginates through uploads playlist, batches video detail calls.
        Returns list of dicts with: video_id, title, description, published_at,
        duration_iso, duration_seconds, category_id, thumbnail_url.
        Stops after max_pages to prevent quota exhaustion.
        """
```

Duration parsing: YouTube returns ISO 8601 duration (e.g. "PT1H23M45S"). Add a helper `_parse_iso_duration(iso: str) -> int | None` that converts to seconds using regex: `r"PT(?:(\d+)H)?(?:(\d+)M)?(?:(\d+)S)?"`.

Category filtering: YouTube categoryId values to skip (non-interview content):
- "10" = Music
- "20" = Gaming
- "22" = People & Blogs (keep -- many interviews are this category)

- "17" = Sports

Add a module-level constant: `SKIP_CATEGORY_IDS = {"10", "20", "17"}` -- these are clearly non-interview content. The handler will use this for filtering.

**Create `src/thinktank/handlers/fetch_youtube_channel.py`:**

Handler following `JobHandler` protocol: `async def handle_fetch_youtube_channel(session: AsyncSession, job: Job) -> None`

Job payload schema: `{"source_id": "uuid-str"}`

Implementation -- mirror fetch_podcast_feed.py structure closely:

a. Extract source_id from payload; load Source; verify approved + active
b. Read global config values via `get_config_value(session, "min_duration_seconds", 600)` and `get_config_value(session, "skip_title_patterns", DEFAULT_SKIP_PATTERNS)` + source overrides via `get_source_filter_config`
c. Add YouTube-specific skip title patterns: `["shorts", "#shorts", "highlights", "clip"]` merged with the source's effective skip patterns
d. Read YouTube API key from system_config: `await get_config_value(session, "youtube_api_key", "")`. If empty, raise ValueError("YouTube API key not configured in system_config").
e. Check rate limit: `await check_and_acquire_rate_limit(session, "youtube", str(job.id))`. If False, raise ValueError("YouTube API rate limited")
f. Extract channel_id from source.external_id. If None, try to parse from source.url (match `youtube.com/channel/(UC[a-zA-Z0-9_-]+)`)
g. Create YouTubeClient(api_key), call `client.fetch_all_channel_videos(channel_id)`
h. For each video returned:
   - Generate URL: `https://www.youtube.com/watch?v={video_id}`
   - Normalize URL via `normalize_url(url)`
   - Layer 1 dedup: check canonical_url exists in Content table
   - Compute fingerprint via `compute_fingerprint(title, published_at, duration_seconds)`
   - Layer 2 dedup: check fingerprint exists
   - Apply filtering:
     - `should_skip_by_duration(duration_seconds, effective_min_duration)` -> "skipped"
     - `should_skip_by_title(title, effective_skip_patterns)` -> "skipped"
     - `video.category_id in SKIP_CATEGORY_IDS` -> "skipped"
     - Otherwise: `status = "cataloged"` (NOT "pending" -- per D-01)
   - Create Content row with content_type="video", source_id, url, canonical_url, fingerprint, title, published_at, duration_seconds, show_name=source.name, status, discovered_at=_now()
   - Collect descriptions dict: {content_id_str: video.description}
i. Flush to get IDs
j. Update source.last_fetched, source.item_count
k. If is_backfill: source.backfill_complete = True
l. Enqueue scan_episodes_for_thinkers job with content_ids (only non-skipped) and descriptions
m. Commit
