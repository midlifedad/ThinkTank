<threat_model>
## Attack Surfaces

1. **XML parsing of podcast:person tags** (new)
   - Threat: XML bomb / billion laughs attack via crafted RSS XML
   - Severity: medium
   - Mitigation: Use `defusedxml.ElementTree` instead of stdlib `xml.etree.ElementTree` if available, OR set explicit size limits on XML input. Since the XML is already fetched and parsed by feedparser (which handles the HTTP layer), the risk is limited to the supplementary parse. Use stdlib ET with a max input size check (reject XML > 10MB).

2. **Content status manipulation via job payload**
   - Threat: Crafted job payload with fake content_ids could promote arbitrary content
   - Severity: low
   - Mitigation: Handler validates content.status == "cataloged" before promotion; only promotes existing content rows. Job payloads are created by internal handlers, not external input.

3. **Unbounded thinker loading in scan handler**
   - Threat: Loading all active thinkers into memory could be expensive with very large thinker tables
   - Severity: low
   - Mitigation: Acceptable for v1 (thinker table is bounded by LLM governance); add pagination if thinker count exceeds 10,000.

**ASVS L1 applicable:** V5 (Input Validation) -- XML input size validation.
</threat_model>

**1. Create `src/thinktank/ingestion/podcast_person_parser.py`:**

Pure-logic module. Single function:

```python
def extract_podcast_persons(xml_content: str) -> dict[str, list[dict]]:
```

- Uses `xml.etree.ElementTree` (stdlib) to parse RSS XML
- Namespace: `PODCAST_NS = "https://podcastindex.org/namespace/1.0"`
- Iterates `root.iter("item")`, for each item finds `guid` element text
- For each item, finds all `{PODCAST_NS}person` elements
- Returns dict mapping guid -> list of person dicts: `{"name": str, "role": str, "group": str, "href": str|None, "img": str|None}`
- Default role is "host" (lowercase), default group is "cast" (lowercase)
- Add XML size guard: if `len(xml_content) > 10_000_000` (10MB), return empty dict with a structlog warning
- No async, no DB, no side effects

**4. Create `src/thinktank/handlers/scan_episodes_for_thinkers.py`:**

Handler following `JobHandler` protocol: `async def handle_scan_episodes_for_thinkers(session: AsyncSession, job: Job) -> None`

Job payload schema:
```python
{
    "content_ids": ["uuid-str", ...],
    "source_id": "uuid-str",
    "descriptions": {"content-id-str": "description text", ...},
    "raw_xml": "optional - raw RSS XML for podcast:person extraction"
}
```

Implementation:
a. Extract payload fields (content_ids, source_id, descriptions, raw_xml)
b. Load source via `session.get(Source, source_id)`
c. Determine if host-owned: query `SourceThinker` for `relationship_type == "host"` rows for this source. If any exist, `is_host_source = True`, collect `host_thinker_ids`
d. Load all active approved thinkers: `select(Thinker).where(Thinker.active == True, Thinker.approval_status == "approved")`. Build `thinker_names = [{"id": t.id, "name": t.name} for t in thinkers]`
e. If raw_xml is provided, call `extract_podcast_persons(raw_xml)` to get person_by_guid dict
f. For each content_id in content_ids:
   - Load content via `session.get(Content, uuid.UUID(content_id_str))`
   - Skip if content is None or content.status != "cataloged"
   - If is_host_source: set `content.status = "pending"`, create ContentThinker rows for each host_thinker_id with role="primary", confidence=10. Continue to next.
   - For guest sources: get description from descriptions dict. Call `match_thinkers_in_text(content.title, description, thinker_names, source_owner_name)`.
   - Also check podcast:person data if available (match person names against thinker_names for confidence=10 matches).
   - If any matches: set `content.status = "pending"`, create ContentThinker rows for each match (checking for existing attribution via `session.get(ContentThinker, (content.id, thinker_id))` before inserting).
   - If no matches: content stays "cataloged" (no action needed).
g. `await session.commit()`
h. Log summary: promoted_count, total scanned, source_id

Import from `src.thinktank.ingestion.name_matcher import match_thinkers_in_text`.
Import from `src.thinktank.ingestion.podcast_person_parser import extract_podcast_persons`.

**1. Create `src/thinktank/handlers/rescan_cataloged_for_thinker.py`:**

Handler following `JobHandler` protocol: `async def handle_rescan_cataloged_for_thinker(session: AsyncSession, job: Job) -> None`

Job payload schema:
```python
{
    "thinker_id": "uuid-str",
    "thinker_name": "Sam Harris"
}
```

Implementation:
a. Extract thinker_id and thinker_name from payload
b. Load thinker via `session.get(Thinker, uuid.UUID(thinker_id))` -- verify exists and is approved
c. Query ALL cataloged content: `select(Content).where(Content.status == "cataloged")` -- the query should use ILIKE for title and description matching:
   ```python
   stmt = select(Content).where(
       Content.status == "cataloged",
       sa.or_(
           Content.title.ilike(f"%{thinker_name}%"),
           # Note: description not stored on Content model directly.
           # Content rows don't have a description column -- descriptions are
           # only available at fetch time via job payloads.
           # So rescan can only match on title.
       )
   )
   ```
   Actually -- looking at the Content model, there is no `description` column. Descriptions come from the RSS feed and are passed via job payload, not stored on Content. So rescan matches on `title` ILIKE only.
d. For each matching content row:
   - Check if ContentThinker already exists for this (content_id, thinker_id) pair -- skip if so
   - Set `content.status = "pending"`
   - Create ContentThinker row with role="guest", confidence=7 (retroactive match, lower confidence than real-time title match of 9)
e. `await session.commit()`
f. Log: promoted_count, thinker_name

Imports: same pattern as scan_episodes_for_thinkers. Use structlog, _now() pattern.
