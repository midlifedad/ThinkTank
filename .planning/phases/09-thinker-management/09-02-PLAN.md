**New endpoints to add:**

1. `GET /admin/thinkers/{thinker_id}` -- Thinker detail page. Load the Thinker by ID (404 if not found). Render thinker_detail.html with thinker data (name, slug, tier, bio, approval_status, active, added_at, category names). The page has HTMX-loaded sections for sources, content, reviews, and a discovery trigger button.

2. `GET /admin/thinkers/{thinker_id}/partials/sources` -- HTMX partial. Query sources where thinker_id matches. Return list of dicts: name, source_type, url, approval_status, active, error_count, last_fetched, item_count. Render thinker_sources.html.

3. `GET /admin/thinkers/{thinker_id}/partials/content` -- HTMX partial. Query content where source_owner_id == thinker_id. Order by published_at DESC, LIMIT 20. Return list of dicts: title, status, published_at, duration_seconds. Render thinker_content.html.

4. `GET /admin/thinkers/{thinker_id}/partials/reviews` -- HTMX partial. Query llm_reviews using raw SQL: `SELECT id, review_type, decision, decision_reasoning, created_at FROM llm_reviews WHERE context_snapshot->>'thinker_id' = :tid ORDER BY created_at DESC LIMIT 20`. This finds all LLM reviews where the context_snapshot JSON contains the thinker_id. Render thinker_reviews.html.

5. `POST /admin/thinkers/{thinker_id}/discover` -- Trigger PodcastIndex discovery. Create a Job row with `job_type="discover_guests_podcastindex"`, `payload={"thinker_id": str(thinker_id)}`, `status="pending"`, `priority=5`. Commit. Re-render the thinker detail page (or return an HTML fragment with a success message "Discovery job queued for {thinker.name}").

6. `GET /admin/thinkers/candidates` -- Candidate queue page. Query CandidateThinker ordered by appearance_count DESC. Return candidates list. Render candidate_queue.html as a full section (not a standalone page -- embed in the thinkers template or create a simple wrapper). Actually, make this a standalone page at /admin/thinkers/candidates that extends base.html.

   **IMPORTANT:** This route MUST be registered BEFORE the `/{thinker_id}` route to avoid FastAPI interpreting "candidates" as a thinker_id UUID. Either place it above the detail route in the file, or use a separate prefix pattern.

7. `POST /admin/thinkers/candidates/{candidate_id}/promote` -- Promote a candidate. Accept Form fields: reason (str). Steps:
   a. Load the CandidateThinker by ID (404 if not found)
   b. Create a new Thinker: name=candidate.name, slug from name, tier=3 (default for promoted), bio=f"Promoted from candidate. Reason: {reason}", approval_status="awaiting_llm"
   c. Create a Job for llm_approval_check on the new thinker
   d. Update candidate: status="promoted", thinker_id=new_thinker.id, reviewed_by="admin", reviewed_at=now
   e. Commit, re-render the candidate_queue.html partial with success message

8. `POST /admin/thinkers/candidates/{candidate_id}/reject` -- Reject a candidate. Accept Form fields: reason (str). Update candidate: status="rejected", reviewed_by="admin", reviewed_at=now. Add a note to reviewed_by or use a simple approach. Commit, re-render candidate_queue.html with success.
