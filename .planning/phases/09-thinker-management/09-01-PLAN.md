**Endpoints to create:**

1. `GET /admin/thinkers/` -- Render thinkers.html page (like config_page pattern)

2. `GET /admin/thinkers/partials/list` -- HTMX partial returning thinker table. Accept query params:
   - `q` (str, optional) -- text search on thinker name using ILIKE `%q%`
   - `tier` (int, optional) -- filter by tier
   - `active` (str, optional) -- "true"/"false"/"all" filter
   Query: `select(Thinker)` with optional `.where(Thinker.name.ilike(f"%{q}%"))` and tier/active filters. Also join to count sources per thinker using a subquery: `select(Source.thinker_id, func.count(Source.id).label("source_count")).group_by(Source.thinker_id).subquery()`. Also load category names via the `categories` relationship (already lazy="selectin" on Thinker model). Return thinker_list.html with thinkers list (each dict: id, name, slug, tier, active, approval_status, source_count, category_names as comma-separated string).

3. `GET /admin/thinkers/partials/add-form` -- Returns the add form partial (just renders the template with list of all categories from DB for the category multiselect)

4. `POST /admin/thinkers/add` -- Create a new thinker. Accept Form fields: name (str), tier (int), bio (str, default ""), category_ids (list of UUID strings from multi-select). Steps:
   a. Generate slug from name: `name.lower().replace(" ", "-")` stripped of non-alphanumeric-dash chars
   b. Create Thinker with approval_status="awaiting_llm", active=True
   c. For each category_id, create a ThinkerCategory row with relevance=5 (default)
   d. Create a Job row: `job_type="llm_approval_check"`, `payload={"entity_type": "thinker", "entity_id": str(thinker.id)}`, `status="pending"`, `priority=5`
   e. Flush, commit, then re-render the thinker_list.html partial (so the HTMX swap shows the new thinker in the list). Include a `success` message: "Thinker '{name}' added. LLM approval queued."

5. `GET /admin/thinkers/{thinker_id}/edit` -- Return the edit form partial for a specific thinker. Load the thinker + all categories, pre-select current categories.

6. `POST /admin/thinkers/{thinker_id}/edit` -- Update thinker. Accept Form fields: name, tier, bio, active (checkbox, "on"/"off"), category_ids. Update thinker fields, delete existing ThinkerCategory rows for this thinker, re-insert new ones. Commit, re-render thinker_list.html with success message.

7. `POST /admin/thinkers/{thinker_id}/toggle-active` -- Toggle thinker.active boolean. Commit, re-render thinker_list.html partial. (Supports THNK-07: deactivate/reactivate without deleting data.)
